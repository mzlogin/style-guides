# GitHub 活动规范

## 提交

* 一次提交最好不要掺杂多种功能更改
* 提交信息不要超过 50 个字符
* 提交信息以一个大写字母开头
* 提交信息应该以更改的文件或者段名称开头
* 提交信息使用[祈使语句][2]
* 提交信息不要用句点结尾
* 如果只是修复了语法错误或者排版，提交信息可以写 `Grammar` 或 `Typo`
* 看一看仓库以前的提交信息，保持一致风格

## 示例

```
Bad: "added my new plugin"
Good: "Plugins: add vim-newplugin"
```
